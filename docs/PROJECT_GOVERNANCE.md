## Purpose

Primary governing document for the James Roman Advisory platform.

## Business Mission

The software serves the practice. The practice does not serve the software.

## Visual Constitution

No future CRM, Office Portal, Admin feature, workflow, integration, or operational tool may reduce:
- visual calmness
- perceived discretion
- perceived competence
- speed

## Architecture

- Next.js
- Vercel
- Supabase
- Postgres
- Auth
- MFA
- Private Storage

## Environments

- prototype
- staging
- production

## Deployment Rules

Production is frozen by default and requires Roman's explicit approval.

## Hotfix Policy

A hotfix may contain:
- one bug
- one root cause
- one deployment objective

Nothing else.

## Final Rule

The goal is not to build software.
The goal is to build trust.
