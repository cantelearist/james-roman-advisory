# Visual Constitution

No future CRM, Office Portal, or Admin feature may reduce:

- visual calmness
- perceived discretion
- perceived competence
- speed

This rule applies to every authenticated surface:

- `/office`
- `/admin`
- future `office.jamesroman.la`
- future `admin.jamesroman.la`

It is a production constraint, not a style preference.

## Enforcement

Before a feature moves from prototype to staging, verify:

- The screen feels quieter, clearer, or more useful than the prior version.
- The interface does not add visual noise, unnecessary cards, generic dashboard clutter, or fake luxury styling.
- The workflow does not expose more information than the user needs for the moment.
- The interaction path is at least as fast as the prior version.
- The page remains fast under expected data volume.
- Copy remains restrained, precise, and confidence-building.

If a feature improves functionality but reduces calmness, discretion, competence, or speed, the feature is not ready.
